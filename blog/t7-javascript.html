Ruby Hashes vs. JavaScript Objects

Aug 1, 2015

    The fundamental difference between Ruby Hashes and JavaScript Objects are the way they are stored and the way they are called upon. While both Ruby Hashes and JavaScript Objects are both stored in curly brackets, Ruby Hashes have a key followed by a '=>' symbol and a value. JavaScript Objects are stored with a colon following the key and square brackets for its value. An example of how both Ruby Hashes and JavaScript Objects are stored:

 

Ruby Hash:

h1 = { "a" => 1, "c" => 2 }

h2 = { 7 => 35, "c" => 2, "a" => 1 }

h3 = { "a" => 1, "c" => 2, 7 => 35 }

h4 = { "a" => 1, "d" => 2, "f" => 35 }

 

 

JavaScript Objects:

var new1 = {

   hungry: false,

   food: ["pizza", "burgers", "pasta", "fries", "sandwich"]

};

 

    Another difference is the way Ruby Hashes and JavaScript Objects are called. Ruby Hashes are called using square brackets with the key inside the square brackets while JavaScript Objects are called using a dot between the values inside of parenthesis. An example of how they would be called:

 

Ruby Hash:

h1 = { "a" => 1, "c" => 2 }

puts h1["a"]

would print out an output of:

1

 

 

JavaScript Objects:

var new1 = {

   food: ["pizza", "burgers", "pasta", "fries", "sandwich"]

};

console.log(new1.food[1])

would print out an output of:

burgers
